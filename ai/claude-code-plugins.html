еңЁдҪҝз”ЁClaude CodeиҝӣиЎҢеӣўйҳҹеҚҸдҪңејҖеҸ‘ж—¶пјҢе·ҘзЁӢеёҲд»¬з»Ҹеёёйқўдёҙиҝҷж ·зҡ„еӣ°еўғпјҡиҙ№еҝғжү“зЈЁзҡ„иҮӘе®ҡд№үSkillsеҸӘиғҪеңЁеҪ“еүҚйЎ№зӣ®дёӯдҪҝз”ЁпјҢзІҫеҝғи®ҫи®Ўзҡ„Hooksе®Ўи®Ўи§„еҲҷж— жі•иҪ»жқҫеҗҢжӯҘз»ҷеӣўйҳҹе…¶д»–жҲҗе‘ҳпјҢдјҳеҢ–иҝҮзҡ„MCPжңҚеҠЎеҷЁй…ҚзҪ®д№ҹж— жі•жү“еҢ…еҲҶеҸ‘гҖӮжҜҸжҚўдёҖдёӘйЎ№зӣ®жҲ–дёҖеҸ°жңәеҷЁпјҢе°ұеҝ…йЎ»йҮҚж–°жүӢеҠЁй…ҚзҪ®дёҖйҒҚгҖӮ

PluginsжӯЈжҳҜдёәдәҶеҪ»еә•и§ЈеҶіиҝҷдёҖз—ӣзӮ№иҖҢиҜһз”ҹзҡ„гҖӮе®ғжҳҜClaude CodeжҸҗдҫӣзҡ„дёҖеҘ—ж ҮеҮҶеҢ–зҡ„еҠҹиғҪжү“еҢ…дёҺеҲҶеҸ‘жңәеҲ¶пјҢе°ҶSkillsгҖҒAgentsгҖҒHooksгҖҒMCPжңҚеҠЎеҷЁгҖҒLSPиҜӯиЁҖжңҚеҠЎеҷЁзӯүжүҖжңүиҮӘе®ҡд№үжү©еұ•ж•ҙеҗҲиҝӣдёҖдёӘиҮӘеҢ…еҗ«зҡ„зӣ®еҪ•пјҢйҖҡиҝҮз»ҹдёҖзҡ„жҸ’д»¶еёӮеңәиҝӣиЎҢе®үиЈ…гҖҒжӣҙж–°е’Ңе…ұдә«гҖӮ

д»Җд№ҲжҳҜPluginsвҖӢ

и®ҫи®Ўзӣ®ж ҮвҖӢ

Pluginsзҡ„ж ёеҝғзӣ®ж ҮжҳҜе®һзҺ°Claude Codeжү©еұ•иғҪеҠӣзҡ„еҸҜжү“еҢ…гҖҒеҸҜзүҲжң¬еҢ–гҖҒеҸҜеҲҶеҸ‘пјҢдё»иҰҒи§ЈеҶід»ҘдёӢеҮ зұ»й—®йўҳпјҡ

  • и·ЁйЎ№зӣ®еӨҚз”ЁпјҡеҗҢдёҖеҘ—Skillsе’ҢHooksеҸҜд»Ҙе®үиЈ…еҲ°еӨҡдёӘйЎ№зӣ®пјҢдёҚеҶҚйҮҚеӨҚзј–еҶҷ
  • еӣўйҳҹеҚҸдҪңпјҡйҖҡиҝҮзүҲжң¬жҺ§еҲ¶пјҲprojectдҪңз”ЁеҹҹпјүжҲ–з§ҒжңүжҸ’д»¶еёӮеңәе°Ҷй…ҚзҪ®е…ұдә«з»ҷж•ҙдёӘеӣўйҳҹ
  • зӨҫеҢәеҲҶеҸ‘пјҡйҖҡиҝҮе®ҳж–№жҲ–з¬¬дёүж–№еёӮеңәеҸ‘еёғжҸ’д»¶пјҢдҫӣжӣҙе№ҝжіӣзҡ„зӨҫеҢәдҪҝз”Ё
  • дҫқиө–йҡ”зҰ»пјҡжҸ’д»¶иҮӘеёҰMCPжңҚеҠЎеҷЁе’Ңи„ҡжң¬пјҢдёҺе®ҝдё»зҺҜеўғи§ЈиҖҰпјҢе®үиЈ…еҚіеҸҜз”Ё
  • е‘ҪеҗҚз©әй—ҙйҡ”зҰ»пјҡжҸ’д»¶еҶ…зҡ„SkillsиҮӘеҠЁд»ҘжҸ’д»¶еҗҚдҪңдёәеүҚзјҖпјҲеҰӮ/my-plugin:helloпјүпјҢйҒҝе…ҚеӨҡжҸ’д»¶й—ҙе‘ҪеҗҚеҶІзӘҒ

name: review
description: Performs comprehensive code review. Use when reviewing code changes, PRs, or analyzing code quality.
---

When reviewing code, check for:
1. Code clarity and readability
2. Error handling completeness
3. Potential security vulnerabilities
4. Test coverage gaps
5. Performance concerns

Provide feedback in a structured format with severity levels: CRITICAL, WARNING, SUGGESTION.

scripts/check.shпјҡ

#!/bin/bash
# Receive hook context from stdin
INPUT=$(cat)
FILE_PATH=$(echo "$INPUT" | jq -r '.tool_input.file_path // empty')

if [[ "$FILE_PATH" == *.ts || "$FILE_PATH" == *.tsx ]]; then
  echo "Running type check on $FILE_PATH..."
  npx tsc --noEmit 2>&1 | head -20
fi

name: query
description: Execute database queries and explain results. Use when user needs to query the database or analyze data.

Use the db-tools MCP server to execute database queries.
Always explain the query logic and result interpretation in plain language.
Warn the user before executing any write operations (INSERT, UPDATE, DELETE).
